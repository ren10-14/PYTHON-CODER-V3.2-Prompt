# Python Coder Prompt V3.2

Промпт промышленного уровня, который превращает любую LLM в **senior Python-разработчика + QA-инженера**.  
Генерирует чистый, типобезопасный, хорошо протестированный код с самоанализом и сводкой для коррекции.

> 🇬🇧 [English version](README-EN.md)

---

## Возможности

- ✅ **Полные аннотации типов** — синтаксис Python 3.10+ (`list[int]`, `dict[str, Any]`, `str | None`)
- ✅ **Валидация входных данных** — проверка и типа контейнера, И типов элементов
- ✅ **Конкретные исключения** — `ValueError`, `TypeError`, `KeyError` и т.д. с понятными сообщениями
- ✅ **Обработка ошибок** — `try-except` где нужно, никакого голого `except`, никаких молчаливых сбоев
- ✅ **Автоматический тестовый набор** — pytest с happy path, граничными случаями и тестами на ошибки
- ✅ **Самоанализ** — секция `[QA-ANALYSIS]` с уровнем уверенности, потенциальными проблемами и соответствием правилам
- ✅ **Сводка для коррекции** — секция `[CORRECTION-SUMMARY]` с предложениями по исправлению и паттернами для запоминания
- ✅ **Соответствие PEP 8** — 4 пробела, snake_case, PascalCase, лимит строк 88 символов
- ✅ **Никаких заглушек и демо-кода** — каждый вывод готов к продакшену
- ✅ **Только стандартная библиотека по умолчанию** — никаких внешних зависимостей, если явно не запрошены
- ✅ **Защита от инъекций** — теги `<user_data>` изолируют инструкции от данных

## Зачем использовать этот промпт?

Большинство промптов для генерации кода выдают **неполный, непротестированный или небезопасный код**.  
Этот промпт обеспечивает **профессиональные стандарты**:

- Каждая функция полностью реализована — никаких `TODO`, `pass`
- Каждая публичная функция валидирует входные данные
- У каждой ошибки понятное сообщение
- Каждый фрагмент кода сопровождается тестами
- Каждый вывод включает **самоанализ** и **извлечённые уроки** для будущих генераций

Это как иметь **senior-разработчика и QA-инженера** в кармане.

## Как использовать

1. **Скопируй промпт**  
   - [`Prompt/PYTHON-CODER-V3.2-EN.md`](Prompt/PYTHON-CODER-V3.2-EN) (английская версия)  
   - [`Prompt/PYTHON-CODER-V3.2-RU.md`](Prompt/PYTHON-CODER-V3.2-RU) (русская версия)

2. **Отправь его LLM** (Claude, GPT-4, DeepSeek и т.д.)  
   Помести описание задачи внутрь тегов `<user_data>`:

```xml
<user_data>
Напиши функцию, которая принимает список чисел и возвращает сумму квадратов чётных чисел
</user_data>
Получи полный ответ с 4 секциями, разделёнными ---:

Основной Python-код (готов к запуску)

Тестовый набор (pytest)

QA-анализ (потенциальные проблемы, граничные случаи, соответствие правилам)

Сводка для коррекции (исправления, паттерны, директива для самокоррекции)

Примеры
Смотри папку Examples/ для полных примеров:

Английские примеры:
Examples/Example-1-EN.md — простая функция с полным пайплайном QA

Examples/Example-2-EN.md — негативный пример (как НЕ надо делать)

Examples/Example-3-EN.md — скрипт с файловым вводом-выводом и мокингом

Русские примеры:
Examples/Examples-RU/Example-1-RU.md

Examples/Examples-RU/Example-2-RU.md

Examples/Examples-RU/Example-3-RU.md

Совместимость
Лучше всего работает с:

Claude (Sonnet, Opus) — XML-теги родные

GPT-4, GPT-4o — markdown-заголовки + JSON mode (при необходимости измени порядок блоков)

DeepSeek — отлично генерирует код

Gemini — с небольшими правками (добавить преамбулу безопасности)

Также работает с локальными моделями (Llama, Mistral) — для маленьких моделей уменьши сложность.
